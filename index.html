Using Postmark to Bypass PHP Mail

Web forms have always been a headache for me. This resulted in an unhealthy reliance on third party solutions to accomplish the task. Along comes HTML5 with its new input types that work pretty well in most modern browsers. Combine this with the jQuery Form Plugin and you've got something that's easy to use and does pretty much everything you need it to; along with giving you the flexibility the third party solutions don't. If you need help, there are plenty of good tutorials out there, for example, Create A Simple HTML5 Contact Form by Landon Miller.

One of the biggest limitations I encountered while building out my HTML5 form was getting my form data into the correct format and sending out an authenticated email. You see, my shared webhost only allows me send email using PHP Mail, and the 'From' email address has to be hosted with them as well. This wasn't an option as I had been using Google Apps for the address attached to this form for a long time and all of my correspondence and contacts reside there.

Fortunately for me, Miller had posted a comment on his tutorial recommending Postmark as an alternative. Postmark provides an API that allows me to bypass PHP Mail and send my authenticated transactional email though them for a small fee (just $1.50 for 1000 emails and the first 1000 emails are free).
