Using Postmark to Bypass PHP Mail

Web forms have always been a headache for me. This resulted in an unhealthy reliance on third party solutions to accomplish the task. Along comes HTML5 with its new input types that work pretty well in most modern browsers. Combine this with the jQuery Form Plugin and you've got something that's easy to use and does pretty much everything you need it to; along with giving you the flexibility the third party solutions don't. If you need help, there are plenty of good tutorials out there. I used Create a Simple HTML 5 Contact Form by Landon Miller which at the time of this writing was returning a 404 (hopefully he'll get this URL working again).

One of the biggest limitations I encountered while building out my HTML5 form was getting my form data into the correct format and sending out an authenticated email. You see, my shared webhost only allows me send email using PHP Mail, and the 'From' email address has to be hosted with them as well. This wasn't an option as I had been using Google Apps for the address attached to this form for sometime and I was loath to swap Gmail's interface for SquirrelMail.

Fortunately for me, Mr. Miller had posted a comment on his tutorial recommending Postmark as an alternative. Postmark provides an API that allows me to bypass PHP Mail and send my authenticated transactional email though them for a small fee (just $1.50 for 100 emails and the first 1000 emails are free).

I checked out Postmark and was eager to give it a try. The path of least resistance was still PHP, so I opted to experiment with the Postmark PHP Class by Markus Hedlund. After adding a CNAME to my DNS and verifying my domain, I started messing around with the script and before long I was able to send a successful test message.

The next step was a bit more difficult, at least for me as it involved passed my form data to the script and parsing the data into the body of the email before sending it out. Despite a good bit of searching, I couldn't find a single tutorial on integrating the jQuery Form Plugin with the Postmark PHP Class, so I was forced to figure it out on my own. Since I'm not a programmer (full disclaimer), I resorted to my old stand by...trail and error. Luckily, this was one of the few times where trail and error actually worked!

Here's where I attempt to step you through what I did. Let's start with process.php from Landon Miller's tutorial:

			// Get Data	
			$name = strip_tags($_POST['name']);
			$email = strip_tags($_POST['email']);
			$message = strip_tags($_POST['message']);

			// Send Message
			mail( "jane@smith.com", "Jane Smith",
			"Name: $name\nEmail: $email\nMessage: $message\n",
			"From: Jane Smith " );

Then let's look at the Postmark PHP script:

	
			// Well, yeah..
			require('Postmark.php');
	
			// Create a "server" in your "rack", then copy it's API key
			define('POSTMARKAPP_API_KEY', 'xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx');
	
			// Create a "Sender signature", then use the "From Email" here.
			// POSTMARKAPP_MAIL_FROM_NAME is optional, and can be overridden
			// with Mail_Postmark::fromName()
			define('POSTMARKAPP_MAIL_FROM_ADDRESS', 'user@example.com');
			define('POSTMARKAPP_MAIL_FROM_NAME', 'Example');
	
			// Create a message and send it
			Mail_Postmark::compose()
				->addTo('jane@smith.com', 'Jane Smith')
				->subject('Subject')
				->messagePlain('Plaintext message')
				->send();

Take the data-parsing variables from process.php:

			// Get Data	
			$name = strip_tags($_POST['name']);
			$email = strip_tags($_POST['email']);
			$message = strip_tags($_POST['message']);

And insert them into the Postmark PHP script:

			// Well, yeah..
			require('Postmark.php');
	
			// Create a "server" in your "rack", then copy it's API key
			define('POSTMARKAPP_API_KEY', 'xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx');
	
			// Create a "Sender signature", then use the "From Email" here.
			// POSTMARKAPP_MAIL_FROM_NAME is optional, and can be overridden
			// with Mail_Postmark::fromName()
			define('POSTMARKAPP_MAIL_FROM_ADDRESS', 'user@example.com');
			define('POSTMARKAPP_MAIL_FROM_NAME', 'Example');
			
			// Get Data	
			$name = strip_tags($_POST['name']);
			$email = strip_tags($_POST['email']);
			$message = strip_tags($_POST['message']);
	
			// Create a message and send it
			Mail_Postmark::compose()
				->addTo('jane@smith.com', 'Jane Smith')
				->subject('Subject')
				->messagePlain('Plaintext message')
				->send();

Finally, you need to get the data into the body of your email using dot separators:

			// Create a message and send it
			Mail_Postmark::compose()
				->addTo('jane@smith.com', 'Jane Smith')
				->subject('Subject')
				->messagePlain('From: ' . $name . "\r\n" . 'Email: ' .  $email . "\r\n" . 'Message: ' . $message)
				->send();
